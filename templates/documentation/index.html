Chapitre 1 : Introduction

L'API SMS est une plateforme complète de gestion et d'envoi de messages SMS qui offre deux interfaces principales :

  • Interface Web
  • API REST :Pour l'intégration avec des applications tierces.

L'API est composé principalement de la :

  • ➔ Gestion des Utilisateurs : Authentification et gestion des profils
  • ➔ Plans SMS : Gestion des crédits et des marques (brand names)
  • ➔ Messages : Création, traitement et envoi des SMS
  • ➔ Abonnements : Recharge des crédits pour les plans

L'utilisateur commence par se connecter sur la plateforme. Ensuite, il fait une demande de plan SMS en choisissant un brand name unique; le plan est initialement en statut PENDING et devient ACTIVE après validation par l'administrateur. Une fois le plan actif, l'administrateur peut créer un abonnement et ajouter des crédits au plan, mettant ainsi à jour la balance disponible pour l'envoi de SMS. Avec un plan actif et des crédits disponibles, l'utilisateur peut alors procéder à l'envoi de ses messages via l'API.

Chapitre 2 : Interface Web Admin

L'interface Web Admin est l'outil principal pour les administrateurs de la plateforme SMS. Elle permet de gérer tous les aspects de l'application.

  • ➔ Se connecter

Pour se connecter, on entre les identifiants que le gestionnaire de l'application nous a donnés.

  • ➔ Interface d'administration

Voici à quoi ressemble une interface d'administration

  • ➔ Consultation utilisateur et Changement de mot de passe

Pour consulter ses données d'utilisateur, on clique sur la ligne utilisateur.

Pour modifier son mot de passe, on peut cliquer sur le mot "ce formulaire" écrit en couleur.

  • ➔ Création et Consultation plan

Pour créer un nouveau plan ou consulter les données des différents plans que l'on possède, on clique sur la ligne Plans.

Quand on crée un nouveau plan, on choisit l'utilisateur auquel on veut attribuer ce plan, on entre un brand name unique qui sera validé et utilisé lors des envois des messages après validation.

  • ➔ Création et Consultation message

Pour créer un nouveau message ou consulter les données des différents messages que l'on a envoyés, on clique sur la ligne Messages.

Pour créer un message, il faut entrer trois données :

  • - le récipient number : qui contiendra les différents numéros auquel on veut envoyer un message
  • - le content : qui contiendra le message que vous voulez envoyer aux correspondants
  • - le brand name : qui contiendra votre nom de marque qui a été validé dans l'url plan

Quand un message n'a pas été envoyé, il peut être supprimé. L'envoi de messages unique permet d'envoyer un contenu à un seul destinataire. L'envoi de messages multiples permet de diffuser le même contenu à plusieurs destinataires simultanément.
